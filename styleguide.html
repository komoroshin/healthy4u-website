Motion Accessibility

Respect User Preferences: Always include prefers-reduced-motion media query

@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

Guidelines:

• Use animations purposefully, not decoratively

• Keep animations subtle and under 500ms for most cases

• Avoid auto-playing animations that loop indefinitely

• No parallax or motion that could trigger motion sickness

Shadow Usage Guidelines

Layering: Higher z-index elements should have stronger shadows

Consistency: Use the same shadow level for similar UI patterns

Hover States: Increase shadow on hover to indicate interactivity

Colored Shadows: Reserve for primary interactions and key moments

Performance: Avoid box-shadow on large scrolling lists; use sparingly

Dark Backgrounds: Reduce shadow opacity or use lighter shadows on dark surfaces

Responsive Behavior

Mobile-First Approach

Core Principle

Design and develop for mobile screens first, then progressively enhance for larger screens. This ensures optimal performance and user experience on all devices.

/* Mobile-first CSS example */
.card {
  width: 100%;
  padding: 20px;
}

@media (min-width: 768px) {
  .card {
    width: 50%;
    padding: 30px;
  }
}

@media (min-width: 1024px) {
  .card {
    width: 33.333%;
    padding: 40px;
  }
}

Design Principles

01. Ease of Use

The interface should be intuitive and accessible, requiring minimal learning curve for users of all technical backgrounds.

02. Natural Communication

Language should be clear, friendly, and human-centered, avoiding medical jargon where possible.

03. Trust & Security

Visual design should communicate safety, professionalism, and certified healthcare management at every touchpoint.

04. Unified Experience

Seamlessly integrate telemedicine, wearable technology, and portal features into one cohesive experience.

How to Use CSS Variables

1. Copy the entire :root block into your main CSS file

2. Reference variables using var(--variable-name) syntax

3. Override for dark mode by redefining variables in a theme class

4. Update globally - changing a variable updates all instances automatically

Benefits:

• Centralized design system management

• Easy theme switching and customization

• Consistent values across entire codebase

• No need for CSS preprocessors

• Native browser support (all modern browsers)
